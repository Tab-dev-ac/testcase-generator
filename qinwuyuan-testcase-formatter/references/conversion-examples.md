# 转换示例

本文档提供了从各种格式转换为秦务员格式的具体示例。

## 示例1：从 testcase-generator-from-prd 的 XMind 格式转换

### 输入（XMind格式）

```markdown
# 订阅消息管理模块测试用例

## 功能测试

### 订阅列表展示

#### TC-F01-001 - 列表页展示所有订阅记录
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入"监控告警中心 > 订阅消息管理 > 订阅管理"
  2. 查看页面展示的订阅列表
- 预期结果: 显示所有订阅配置记录，字段包括订阅名称、订阅应用、订阅方式、订阅人、状态、创建时间、操作

#### TC-F01-002 - 列表分页显示
- 优先级: P1
- 前置条件: 订阅记录超过10条
- 测试步骤:
  1. 进入订阅管理页面
  2. 查看分页组件
  3. 点击"下一页"
  4. 切换每页显示条数为20
- 预期结果: 分页组件正常工作，支持页码切换和每页条数调整

### 新增订阅

#### TC-F02-001 - 新增订阅-填写所有字段保存成功
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 点击"新增订阅"按钮
  2. 填写订阅名称：告警规则订阅
  3. 选择订阅应用：应用安全
  4. 选择订阅方式：微信
  5. 选择订阅人：张三、李四
  6. 点击"保存"按钮
- 预期结果: 保存成功，列表显示新增记录
```

### 输出（秦务员格式）

```markdown
# 项目名：陕西秦务员

## 模块名：订阅消息管理

### 用例名：订阅列表展示

- 操作步骤1：进入"监控告警中心 > 订阅消息管理 > 订阅管理"

- 操作步骤2：查看页面展示的订阅列表

- 验证：显示所有订阅配置记录

### 用例名：列表分页显示

- 操作步骤1：进入订阅管理页面

- 操作步骤2：查看分页组件

- 操作步骤3：点击"下一页"

- 操作步骤4：切换每页显示条数为20

- 验证：分页组件正常工作，支持页码切换和每页条数调整

### 用例名：新增订阅-填写所有字段保存成功

- 操作步骤1：点击"新增订阅"按钮

- 操作步骤3：选择订阅应用：应用安全

- 操作步骤4：选择订阅方式：微信

- 验证：保存成功，列表显示新增记录
```

### 转换逻辑说明

| 原格式 | 转换后 | 说明 |
|--------|--------|------|
| # 订阅消息管理模块测试用例 | # 项目名：陕西秦务员 | 根节点转换为项目名（项目名可配置） |
| ## 功能测试 | ## 模块名：订阅消息管理 | 二级标题转换为模块名 |
| ### 订阅列表展示 | ### 用例名：订阅列表展示 | 三级标题转换为用例名 |
| #### TC-F01-001 | （忽略） | 四级标题的用例编号在转换中不保留 |
| - 优先级: P0 | （忽略或转换为注释） | 优先级信息可选保留 |
| - 前置条件: XXX | （前置步骤） | 如果有前置条件，可作为第一个操作步骤 |
| - 测试步骤: 1. XXX | - 操作步骤1：XXX | 测试步骤转换为操作步骤 |
| 2. XXX | - 操作步骤2：XXX | 每个步骤单独一行 |
| - 预期结果: XXX | - 验证：XXX | 预期结果转换为验证点 |

## 示例2：从 JSON 格式转换

### 输入（JSON格式）

```json
{
  "project": "陕西秦务员",
  "modules": [
    {
      "name": "自动首页",
      "testcases": [
        {
          "name": "常规版切换到长辈版测试",
          "steps": [
            { "action": "点击'长辈版'按钮", "validation": "页面有\"常规版\"按钮" },
            { "action": "点击'常规版'按钮", "validation": "页面有\"长辈版\"按钮" }
          ]
        },
        {
          "name": "点击全部跳转办事页",
          "steps": [
            { "action": "点击'办事'按钮", "validation": "页面有\"立即添加\"" }
          ]
        }
      ]
    },
    {
      "name": "自动搜索",
      "testcases": [
        {
          "name": "首页搜索",
          "steps": [
            { "action": "点击页面中部\"陕西省\"后的输入框", "validation": "" },
            { "action": "搜索框中输入'宗教政策及'", "validation": "页面有'宗教政策及法律法规咨询'" }
          ]
        }
      ]
    }
  ]
}
```

### 输出（秦务员格式）

```markdown
# 项目名：陕西秦务员

## 模块名：自动首页

### 用例名：常规版切换到长辈版测试

- 操作步骤1：点击'长辈版'按钮

- 验证：页面有"常规版"按钮

- 操作步骤2：点击'常规版'按钮

- 验证：页面有"长辈版"按钮

### 用例名：点击全部跳转办事页

- 操作步骤1：点击'办事'按钮

- 验证：页面有"立即添加"

## 模块名：自动搜索

### 用例名：首页搜索

- 操作步骤1：点击页面中部"陕西省"后的输入框

- 操作步骤2：搜索框中输入'宗教政策及'

- 验证：页面有'宗教政策及法律法规咨询'
```

### 转换逻辑说明

1. **项目名**: 从 `project` 字段读取
2. **模块**: 遍历 `modules` 数组，每个元素生成一个 `## 模块名`
3. **用例**: 遍历每个模块的 `testcases` 数组，生成 `### 用例名`
4. **步骤**: 遍历每个用例的 `steps` 数组，生成 `- 操作步骤N：`
5. **验证点**: 如果 `validation` 字段非空，生成 `- 验证：`

## 示例3：复杂用例转换

### 输入（包含多个验证点的用例）

```markdown
#### TC-EX-001 - 搜索结果加入收藏后取消
- 优先级: P1
- 测试步骤:
  1. 点击页面中部"陕西省"后的输入框
  2. 搜索框中输入'宗教政策及'
  3. 点击第一个"宗教政策及法律法规咨询"
  4. 点击右上角的'...'的图标
  5. 点击'收藏'按钮
  6. 点击左上角的返回'<'图标
  7. 点击'取消'按钮
  8. 点击'我的'按钮
  9. 点击'我的收藏'按钮
  10. 点击"宗教政策及法律法规咨询"后面的'取消收藏'按钮
- 预期结果:
  - 步骤2后：页面有'宗教政策及法律法规咨询'
  - 步骤10后：页面有'没有收藏内容'的字样
```

### 输出（秦务员格式）

```markdown
### 用例名：搜索结果加入收藏后取消

- 操作步骤1：点击页面中部"陕西省"后的输入框

- 操作步骤2：搜索框中输入'宗教政策及'

- 验证：页面有'宗教政策及法律法规咨询'

- 操作步骤3：点击第一个"宗教政策及法律法规咨询"

- 操作步骤4：点击右上角的'...'的图标

- 操作步骤5：点击'收藏'按钮

- 操作步骤6：点击左上角的返回'<'图标

- 操作步骤7：点击'取消'按钮

- 操作步骤8：点击'我的'按钮

- 操作步骤9：点击'我的收藏'按钮

- 操作步骤10：点击"宗教政策及法律法规咨询"后面的'取消收藏'按钮

- 验证：页面有'没有收藏内容'的字样
```

### 转换逻辑说明

当原始格式中有多个验证点时：

1. **每个步骤后的验证**: 直接转换为该步骤后的 `- 验证：`
2. **汇总的验证点**: 按照描述中的步骤编号，插入到对应步骤后
3. **最终验证**: 如果是最终状态的验证，放在所有步骤之后

## 示例4：分组策略

### 输入（按测试类型分组）

```markdown
# 测试用例

## 功能测试

### 功能点1
#### TC-F01-001 - 功能测试用例
- 测试步骤: 1. 步骤1

## 业务流程测试

### 主流程
#### TC-BP-001 - 流程测试用例
- 测试步骤: 1. 步骤1

## 权限测试

### 安全管理员权限
#### TC-PERM-001 - 权限测试用例
- 测试步骤: 1. 步骤1
```

### 输出（按功能区域分组 - 推荐方式）

```markdown
# 项目名：陕西秦务员

## 模块名：[根据功能点推断的模块名]

### 用例名：功能测试用例

- 操作步骤1：步骤1

## 模块名：[根据流程推断的模块名]

### 用例名：流程测试用例

- 操作步骤1：步骤1

## 模块名：[根据权限范围推断的模块名]

### 用例名：权限测试用例

- 操作步骤1：步骤1
```

### 分组策略说明

| 原始分组 | 推荐转换 | 说明 |
|----------|----------|------|
| 按测试类型（功能/流程/权限） | 按功能区域（首页/搜索/办事） | 秦务员格式更倾向于按功能模块组织 |
| 按优先级（P0/P1/P2） | 不推荐，应混合展示 | 优先级可在用例名中标注 |
| 按页面/功能 | **推荐** | 符合秦务员格式规范 |

### 模块名推断规则

```
关键词 → 模块名
首页、启动、导航 → 自动首页
搜索、查询、检索 → 自动搜索
办事、申请、办理 → 自动办事页
收藏、我的 → 个人中心
设置、配置 → 设置管理
消息、通知 → 消息通知
```

## 示例5：处理特殊字段

### 输入（包含前置条件）

```markdown
#### TC-F01-001 - 需要登录的用例
- 前置条件: 安全管理员登录系统，拥有订阅管理权限
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击新增按钮
- 预期结果: 进入新增页面
```

### 输出（前置条件转换为第一步）

```markdown
### 用例名：需要登录的用例

- 操作步骤1：安全管理员登录系统，拥有订阅管理权限

- 操作步骤2：进入订阅管理页面

- 操作步骤3：点击新增按钮

- 验证：进入新增页面
```

### 转换逻辑

- **前置条件**: 转换为第一个操作步骤（标记为前置条件）
- **优先级**: 可选地在用例名中标注，如 `### 用例名：XXX [P0]`
- **标签/状态**: 通常不保留，除非有特殊需求

## 转换注意事项

1. **步骤编号重置**: 每个用例内的步骤编号从1开始，不要延续上一个用例的编号
2. **验证点位置**: 验证点应该紧跟在对应的步骤之后
3. **空行使用**: 模块之间、用例之间建议使用空行分隔，提高可读性
4. **特殊字符处理**: 引号、特殊符号需要正确转义
5. **中英文标点**: 确保使用中文冒号 `：` 而非英文冒号 `:`
