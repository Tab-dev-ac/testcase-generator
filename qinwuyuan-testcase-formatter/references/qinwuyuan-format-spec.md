# Qinwuyuan (秦务员) 测试用例格式规范

## 文档概述

本文档定义了秦务员项目测试用例的标准格式，用于向下游测试团队交付测试用例文档。

## 格式结构

```
项目名 (Level 1)
├── 模块名 (Level 2)
│   ├── 用例名 (Level 3)
│   │   ├── 操作步骤1
│   │   ├── 验证点
│   │   ├── 操作步骤2
│   │   └── 验证点
│   └── 用例名2
└── 模块名2
```

## 详细格式规范

### 1. 项目名

**格式**: `# 项目名：[项目名称]`

**规则**:
- 使用一级标题 (`#`)
- 前缀固定为 "项目名："
- 项目名称后不加标点符号
- 一个文档只包含一个项目

**示例**:
```markdown
# 项目名：陕西秦务员
```

### 2. 模块名

**格式**: `## 模块名：[模块名称]`

**规则**:
- 使用二级标题 (`##`)
- 前缀固定为 "模块名："
- 模块名称简洁明了，通常按功能区域划分
- 模块之间建议空一行分隔（可选）

**常见模块命名**:
- 自动首页
- 自动搜索
- 自动办事页
- 用户中心
- 设置管理
- 消息通知

**示例**:
```markdown
## 模块名：自动首页

## 模块名：自动搜索
```

### 3. 用例名

**格式**: `### 用例名：[测试用例名称]`

**规则**:
- 使用三级标题 (`###`)
- 前缀固定为 "用例名："
- 用例名称应清晰描述测试目标
- 用例之间建议空一行分隔（可选）

**用例命名规范**:
- 使用"功能+场景+测试"的结构
- 示例:
  - "常规版切换到长辈版测试"
  - "点击全部跳转办事页"
  - "首页搜索"
  - "搜索结果加入收藏后取消"

**示例**:
```markdown
### 用例名：常规版切换到长辈版测试

### 用例名：点击全部跳转办事页
```

### 4. 操作步骤

**格式**: `- 操作步骤N：[操作描述]`

**规则**:
- 使用无序列表 (`-`)
- 前缀固定为 "操作步骤"
- 步骤编号使用数字 (1, 2, 3...)
- 步骤后使用中文冒号 `：`
- 冒号后加一个空格
- 操作描述清晰、具体、可执行
- 步骤在同一用例内连续编号

**操作描述编写指南**:
- 使用动宾结构：点击XXX按钮、输入XXX、选择XXX
- 引用页面元素时使用单引号：点击'长辈版'按钮
- 描述具体位置：点击页面中部"陕西省"后的输入框
- 包含必要的前置动作：点击'我的'按钮，然后点击'我的收藏'按钮

**示例**:
```markdown
- 操作步骤1：点击'长辈版'按钮

- 操作步骤2：点击'常规版'按钮

- 操作步骤3：点击页面中部"陕西省"后的输入框

- 操作步骤4：搜索框中输入'宗教政策及'
```

### 5. 验证点

**格式**: `- 验证：[验证内容]`

**规则**:
- 使用无序列表 (`-`)
- 前缀固定为 "验证："
- 使用中文冒号 `：`
- 冒号后加一个空格
- 验证点可以是：
  - 页面元素的显示：页面有"常规版"按钮
  - 页面文本的存在：页面有'没有收藏内容'的字样
  - 数据的展示：页面有'全省督办'字样
- 验证点可以出现在：
  - 每个操作步骤后（推荐）
  - 多个步骤后汇总

**验证点编写指南**:
- 使用"页面有XXX"的句式
- 引用元素时使用单引号或双引号：页面有'宗教政策及法律法规咨询'
- 描述明确的验证目标
- 避免模糊表述

**示例**:
```markdown
- 验证：页面有"常规版"按钮

- 验证：'页面有"长辈版"按钮'

- 验证：页面有'没有收藏内容'的字样

- 验证：页面有'全省督办'字样
```

## 格式检查清单

交付前请检查以下项目：

- [ ] 项目名格式正确：`# 项目名：XXX`
- [ ] 模块名格式正确：`## 模块名：XXX`
- [ ] 用例名格式正确：`### 用例名：XXX`
- [ ] 操作步骤使用数字编号：`操作步骤1`、`操作步骤2`...
- [ ] 步骤编号连续，无跳号
- [ ] 使用中文冒号 `：` 而非英文冒号 `:`
- [ ] 冒号后有空格
- [ ] 验证点格式正确：`- 验证：XXX`
- [ ] 页面元素引用使用引号（单引号或双引号）
- [ ] 操作描述清晰具体，可执行
- [ ] 验证点明确，可验证
- [ ] 一个文档只包含一个项目

## 常见错误

### 错误1：使用英文冒号

❌ 错误：
```markdown
- 操作步骤1: 点击'长辈版'按钮
```

✅ 正确：
```markdown
- 操作步骤1：点击'长辈版'按钮
```

### 错误2：缺少空格

❌ 错误：
```markdown
- 操作步骤1：点击'长辈版'按钮
```

✅ 正确：
```markdown
- 操作步骤1：点击'长辈版'按钮
```

### 错误3：步骤编号不连续

❌ 错误：
```markdown
- 操作步骤1：点击'长辈版'按钮
- 操作步骤3：点击'常规版'按钮
```

✅ 正确：
```markdown
- 操作步骤1：点击'长辈版'按钮
- 操作步骤2：点击'常规版'按钮
```

### 错误4：标题层级错误

❌ 错误：
```markdown
### 项目名：陕西秦务员
#### 模块名：自动首页
##### 用例名：常规版切换到长辈版测试
```

✅ 正确：
```markdown
# 项目名：陕西秦务员
## 模块名：自动首页
### 用例名：常规版切换到长辈版测试
```

## 格式变体

### 变体1：验证点汇总

不是每个步骤都有验证点，可以在多个步骤后汇总：

```markdown
### 用例名：多步骤操作

- 操作步骤1：点击'按钮A'

- 操作步骤2：点击'按钮B'

- 操作步骤3：点击'按钮C'

- 验证：页面显示成功提示
```

### 变体2：无验证点

某些步骤可能不需要验证点，省略即可：

```markdown
### 用例名：简单导航

- 操作步骤1：点击'首页'按钮

- 操作步骤2：点击'办事'按钮

- 验证：进入办事页面
```

## 交付标准

向下游交付的测试用例文档应满足：

1. **格式规范**: 严格遵守本文档格式要求
2. **内容完整**: 包含所有必要的操作步骤和验证点
3. **可执行性**: 每个步骤都可以被测试人员独立执行
4. **可验证性**: 每个验证点都有明确的验证目标
5. **可维护性**: 用例命名规范，易于后续维护和更新
