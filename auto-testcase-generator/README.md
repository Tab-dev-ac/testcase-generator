# Automated Test Case Generator

## 简介

这是一个自动化测试用例生成工具，能够将XMind格式的测试用例转换为标准化的格式（包括秦务员格式）。该工具确保完整提取所有测试用例，并智能处理各种字段。

## 核心功能

### 1. 完整提取

- ✅ 提取所有测试类型：功能、业务流程、权限、界面、异常、验收
- ✅ 不会遗漏任何测试用例
- ✅ 正确处理多级标题结构

### 2. 智能过滤

- ✅ 自动去除不在模板中的字段（优先级、状态、用例编号）
- ✅ 智能处理前置条件（可执行→步骤，状态描述→去除）
- ✅ 字段过滤规则可配置

### 3. 灵活分组

- ✅ 支持按功能模块分组
- ✅ 支持按测试类型分组
- ✅ 支持自定义分组映射
- ✅ 支持扁平化列表

### 4. 格式标准化

- ✅ 自动重置步骤编号
- ✅ 使用中文冒号 `：`
- ✅ 符合秦务员格式规范
- ✅ 可生成自定义格式

## 支持的输入格式

### XMind格式（推荐）

由 `testcase-generator-from-prd` 技能生成：

```markdown
# 订阅消息管理模块测试用例

## 功能测试

### F01 - 订阅列表展示

#### TC-F01-001 - 列表页展示所有订阅记录
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入"监控告警中心 > 订阅消息管理 > 订阅管理"
  2. 查看页面展示的订阅列表
- 预期结果: 显示所有订阅配置记录
- 状态: 待执行
```

## 支持的输出格式

### 秦务员格式（默认）

```markdown
# 项目名：陕西秦务员

## 模块名：订阅消息管理

### 用例名：列表页展示所有订阅记录

- 操作步骤1：安全管理员登录系统

- 操作步骤2：进入"监控告警中心 > 订阅消息管理 > 订阅管理"

- 操作步骤3：查看页面展示的订阅列表

- 验证：显示所有订阅配置记录
```

### 自定义格式

可根据项目需求生成自定义格式。

```bash

# 2. 使用 auto-testcase-generator 技能转换

```

### 命令行使用（如果有CLI）

```bash
auto-testcase-generator \
  --input testcases-xmind.md \
  --output testcases-qinwuyuan.md \
  --format qinwuyuan
```

## 特性详解

### 完整性保证

**确保不遗漏任何测试用例**：

1. **遍历所有测试类型**
   - 功能测试
   - 业务流程测试
   - 权限测试
   - 界面测试
   - 异常测试
   - 验收测试

2. **提取每个类型下的所有功能点**
   - 如：F01、F02、F03...

3. **提取每个功能点下的所有测试用例**
   - 如：TC-F01-001、TC-F01-002...

4. **数量验证**
   - 输入测试用例数量 = 输出测试用例数量

### 智能分组

**多种分组策略**：

1. **按功能模块**（推荐）
   ```
   模块: 订阅管理
   ├── 列表展示
   ├── 新增订阅
   ├── 编辑订阅
   └── 删除订阅
   ```

2. **按测试类型**
   ```
   模块: 功能测试
   ├── 所有功能测试用例

   模块: 业务流程测试
   ├── 所有业务流程测试用例
   ```

3. **扁平化列表**
   ```
   模块: 订阅消息管理
   ├── 测试用例1
   ├── 测试用例2
   └── ...
   ```

### 字段处理

**前置条件处理规则**：

| 前置条件 | 处理方式 | 示例 |
|----------|----------|------|
| 安全管理员登录系统 | 转换为步骤1 | ✅ `- 操作步骤1：安全管理员登录系统` |
| 系统正常运行 | 去除 | ❌ 不生成 |
| 环境已就绪 | 去除 | ❌ 不生成 |
| 订阅记录超过10条 | 转换为步骤1 | ✅ `- 操作步骤1：订阅记录超过10条` |

**字段过滤规则**：

- ✅ 保留：项目名、模块名、用例名、操作步骤、验证点
- ❌ 去除：优先级、状态、用例编号、其他元数据

## 验证清单

生成测试用例后，使用以下清单验证：

### 数量验证

- [ ] 输入测试用例数量 = 输出测试用例数量

- [ ] 模块名格式正确：`## 模块名：XXX`

## 故障排除

### 问题：测试用例不全

**可能原因**：
1. 某些测试类型未被识别
2. 模块分组过于激进
3. 解析错误

**解决方案**：
1. 检查测试类型名称是否完全匹配
2. 尝试使用扁平化列表分组
3. 查看详细的提取日志

### 问题：步骤编号不正确

**可能原因**：
1. 前置条件未正确处理
2. 步骤偏移量计算错误

**解决方案**：
1. 确认前置条件是否转换为步骤
2. 检查 step_offset 的计算逻辑

### 问题：模块分组不合理

**解决方案**：
1. 尝试不同的分组策略
2. 使用自定义分组映射
3. 保留原始文档结构
