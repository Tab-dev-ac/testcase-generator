# Automated Test Case Generator

## Overview

Convert test case metadata from XMind format (generated by testcase-generator-from-prd) into standardized test case documents. Supports multiple output formats including Qinwuyuan format, with intelligent field filtering and comprehensive test case extraction.

## Workflow

```
PRD Document
    ↓
testcase-generator-from-prd
    ↓
XMind Format Test Cases
    ↓
auto-testcase-generator
    ↓
Standardized Format (Qinwuyuan/Custom)
    ↓
Downstream Delivery
```

## Quick Start

### Basic Usage

```bash
# 1. Read XMind format test cases
read /path/to/testcases-xmind.md

# 2. Convert to target format (default: Qinwuyuan)
# The generator will:
# - Extract all test cases including all test types (功能、业务流程、权限、界面、异常、验收)
# - Filter out unnecessary fields (优先级、状态、用例编号等)
# - Convert preconditions to steps when executable
# - Generate standardized output

# 3. Save the result
write output-testcases.md
```

## Field Filtering Rules

### Must Keep Fields ✅

| Field | Format | Example |
|-------|--------|---------|
| 项目名 | `# 项目名：XXX` | `# 项目名：陕西秦务员` |
| 模块名 | `## 模块名：XXX` | `## 模块名：订阅消息管理` |
| 用例名 | `### 用例名：XXX` | `### 用例名：列表页展示所有订阅记录` |
| 操作步骤 | `- 操作步骤N：XXX` | `- 操作步骤1：进入订阅管理页面` |
| 验证点 | `- 验证：XXX` | `- 验证：显示所有订阅记录` |

### Must Remove Fields ❌

| Field | Reason |
|-------|--------|
| 优先级 (P0/P1/P2) | Not in template |
| 状态 (待执行/通过/失败) | Not in template |
| 用例编号 (TC-F01-001) | Not in template |
| 其他元数据 (创建时间、创建人等) | Not in template |

### Preconditions Handling 🔄

**Rule**: Convert executable preconditions to steps, remove status descriptions.

**Executable → Convert to Step**:
```
Input: - 前置条件: 安全管理员登录系统
Output: - 操作步骤1：安全管理员登录系统
```

**Status Description → Remove**:
```
Input: - 前置条件: 系统正常运行
Output: (Removed)
```

## Test Case Extraction

### Full Extraction Algorithm

```python
# Extract ALL test cases from all test types
test_types = [
    "功能测试",
    "业务流程测试",
    "权限测试",
    "界面测试",
    "异常测试",
    "验收测试"
]

all_testcases = []
for test_type in test_types:
    # Find all test cases under each test type
    testcases = extract_testcases(test_type)
    all_testcases.extend(testcases)

# Group by functional module
grouped_testcases = group_by_module(all_testcases)

# Generate output
generate_output(grouped_testcases)
```

### Module Grouping Strategy

**Priority Order**:
1. **Preserve original structure** if functional modules exist
2. **Group by test type** if no clear functional modules
3. **Flat list** if neither applies

### Step Numbering

```javascript
// Reset step number for each testcase
for (testcase of allTestcases) {
  let stepNumber = 1;
  for (step of testcase.steps) {
    output += `- 操作步骤${stepNumber}：${step.action}\n`;
    if (step.validation) {
      output += `- 验证：${step.validation}\n`;
    }
    stepNumber++;
  }
}
```

## Troubleshooting

### Issue: Missing Test Cases

**Symptom**: Output has fewer test cases than input

**Possible Causes**:
1. Test type not recognized
2. Module grouping too aggressive
3. Filter rule too strict

**Solutions**:
1. Check if test type names match exactly
2. Use flat list grouping if module grouping fails
3. Review filter rules

### Issue: Wrong Step Numbers

**Symptom**: Step numbers don't reset for each testcase

**Solution**: Ensure stepNumber is reset at start of each testcase

### Issue: Preconditions Not Handled

**Symptom**: Preconditions either missing or not converted

**Solution**: Check precondition handling logic:
- Executable → Convert to step
- Status → Remove
- Ambiguous → Ask user
