# XMind 输出格式

XMind 支持导入 Markdown 格式的缩进列表来创建思维导图。

## Markdown 格式规则

- 使用缩进表示层级关系（2 空格或 1 tab）
- 使用 `-` 或 `*` 表示列表项
- 可以使用 `[x]` 表示已完成，`[ ]` 表示未完成

## 示例：订阅管理模块测试用例

```markdown
# 订阅消息管理模块测试用例

## 功能测试

### 订阅列表展示

#### TC-F01-001 - 列表页展示所有订阅记录
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入"监控告警中心 > 订阅消息管理 > 订阅管理"
  2. 查看页面展示的订阅列表
- 预期结果: 显示所有订阅配置记录，字段包括订阅名称、订阅应用、订阅方式、订阅人、状态、创建时间、操作
- 状态: 待执行

#### TC-F01-002 - 列表分页显示
- 优先级: P1
- 前置条件: 订阅记录超过10条
- 测试步骤:
  1. 进入订阅管理页面
  2. 查看分页组件
  3. 点击"下一页"
  4. 切换每页显示条数为20
- 预期结果: 分页组件正常工作，支持页码切换和每页条数调整
- 状态: 待执行

### 新增订阅

#### TC-F02-001 - 新增订阅-填写所有字段保存成功
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击"新增订阅"按钮
  3. 填写订阅名称（如："测试订阅"）
  4. 选择订阅应用（如："应用A"）
  5. 勾选订阅方式（如："钉钉"）
  6. 选择订阅人（如："张三"）
  7. 选择状态为"启用"
  8. 填写备注（选填）
  9. 点击"保存"按钮
- 预期结果: 保存成功，弹窗关闭，列表显示新增的订阅记录，提示"保存成功"
- 状态: 待执行

#### TC-F02-002 - 新增订阅-订阅名称为空时提示错误
- 优先级: P1
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击"新增订阅"按钮
  3. 订阅名称留空
  4. 填写其他必填字段
  5. 点击"保存"按钮
- 预期结果: 提示"订阅名称不能为空"，不提交表单
- 状态: 待执行

#### TC-F02-003 - 新增订阅-订阅名称超过50字符时提示错误
- 优先级: P2
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击"新增订阅"按钮
  3. 订阅名称输入51个字符
  4. 填写其他必填字段
  5. 点击"保存"按钮
- 预期结果: 提示"订阅名称不能超过50个字符"，不提交表单
- 状态: 待执行

## 业务流程测试

### 主流程

#### TC-BP-001 - 完整的订阅配置流程
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击"新增订阅"
  3. 填写完整的订阅配置
  4. 保存订阅
  5. 在列表中找到新增的订阅
  6. 点击"编辑"修改配置
  7. 保存修改
  8. 点击"查看"查看详情
- 预期结果: 每个操作都成功执行，数据正确保存和展示
- 状态: 待执行

### 分支流程

#### TC-BP-002 - 订阅人选择流程
- 优先级: P1
- 前置条件: 安全管理员登录系统，打开新增订阅弹窗
- 测试步骤:
  1. 点击"订阅人"选择器
  2. 在搜索框输入"张三"
  3. 勾选搜索结果中的用户
  4. 继续搜索"李四"
  5. 勾选用户
  6. 点击确认关闭选择器
- 预期结果: 订阅人选择器显示已选择2人（张三、李四），以标签形式展示
- 状态: 待执行

## 权限测试

### 安全管理员权限

#### TC-PERM-001 - 安全管理员拥有完整操作权限
- 优先级: P0
- 前置条件: 安全管理员登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 检查是否显示"新增订阅"按钮
  3. 检查列表每行的操作列是否显示"编辑"、"删除"按钮
  4. 检查是否支持批量删除
- 预期结果: 安全管理员拥有新增、编辑、删除、查看、批量删除的完整权限
- 状态: 待执行

### 应用安全负责人权限

#### TC-PERM-002 - 应用安全负责人只有查看权限
- 优先级: P0
- 前置条件: 应用安全负责人登录系统
- 测试步骤:
  1. 进入订阅管理页面
  2. 检查是否不显示"新增订阅"和"批量删除"按钮
  3. 检查列表每行的操作列是否只显示"查看"按钮
- 预期结果: 应用安全负责人只能查看列表和详情，无法进行增删改操作
- 状态: 待执行

## 异常测试

### 网络异常

#### TC-EX-001 - 网络请求失败时提示错误
- 优先级: P1
- 前置条件: 模拟网络断开或服务器错误
- 测试步骤:
  1. 进入订阅管理页面
  2. 点击"新增订阅"
  3. 填写表单
  4. 点击"保存"
- 预期结果: 提示"网络错误，请稍后重试"，弹窗不关闭，可以重新尝试
- 状态: 待执行

### 数据异常

#### TC-EX-002 - 订阅名称重复时提示错误
- 优先级: P1
- 前置条件: 系统中已存在"测试订阅"
- 测试步骤:
  1. 点击"新增订阅"
  2. 输入订阅名称"测试订阅"
  3. 填写其他必填字段
  4. 点击"保存"
- 预期结果: 提示"订阅名称已存在"，停留在表单页
- 状态: 待执行
```

## 导入 XMind 的方法

1. 打开 XMind 软件
2. 点击"文件" -> "导入" -> "Markdown"
3. 选择生成的 .md 文件
4. XMind 会自动将其转换为思维导图

## XMind 节点说明

- **根节点**: 测试用例（模块名称）
- **一级节点**: 测试分类（功能测试、流程测试、权限测试、异常测试）
- **二级节点**: 功能点或具体场景
- **三级节点**: 具体测试用例（TC-ID）
- **四级节点**: 测试用例的详细属性（优先级、前置条件、测试步骤、预期结果、状态）

## 测试用例 ID 规则

- **TC**: Test Case（测试用例）
- **模块标识**: 
  - F: Function（功能测试）
  - BP: Business Process（业务流程测试）
  - PERM: Permission（权限测试）
  - UI: User Interface（界面测试）
  - EX: Exception（异常测试）
- **序号**: 三位数字（001, 002, ...）

示例:
- `TC-F01-001`: 功能测试-列表展示-第1个用例
- `TC-BP-001`: 业务流程测试-主流程-第1个用例
- `TC-EX-001`: 异常测试-网络异常-第1个用例
