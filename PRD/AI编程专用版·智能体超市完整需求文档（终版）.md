# 三、业务核心设计

## 3.1 角色权限定义

- **普通业务用户**：浏览智能体目录、查看详情、提交使用申请、查看个人申请/授权记录、释放个人授权

- **部门审核员**：处理本部门用户申请、填写审核意见、查看部门内申请与授权数据

- **系统管理员**：全平台权限，可审核所有申请、管理全量授权、配置智能体基础信息、处理异常授权

## 3.2 核心业务流程

1. 用户浏览智能体目录→查看智能体详情→提交使用申请

2. 审核员/管理员审核申请（通过/驳回）→通过则自动生成授权信息与密钥

3. 用户获取授权密钥，在有效期内调用智能体接口

4. 支持授权续期、手动回收、过期自动失效，全程留痕可追溯

## 3.3 全局状态枚举（前后端共用）

```TypeScript
// 全局状态枚举，统一前后端字段与取值

  APPROVED: '已通过',       // 申请通过/授权生效
  REJECTED: '已驳回',       // 申请驳回
  EXPIRED: '已过期',        // 授权到期
  REVOKED: '已回收'         // 手动回收授权
} as const;
```

## 6.3 业务规则与异常处理

- **申请校验**：同一用户对单个智能体仅1条待审核申请，授权有效期≤365天，必填项缺失直接拦截

- **审核权限**：审核员仅处理本部门申请，管理员处理全平台申请，驳回需填写非空理由

- **授权失效**：到期自动失效、手动回收立即失效、用户离职批量失效，失效密钥作废

- **日志留存**：全流程操作留痕，记录操作人、时间、IP，日志保留180天

- **异常处理**：防重复提交、事务回滚、敏感数据脱敏、超时重试机制
