# 运维安全管理系统 - 订阅消息管理模块 PRD

## 1. 需求概述

### 1.1 需求背景
随着运维安全管理系统中安全风险告警场景的不断增加，原有的风险通报机制由后端配置固定，无法满足灵活的业务需求。现需要新增前端管理界面，实现安全风险订阅功能的可视化配置，支持安全管理员灵活配置订阅规则、订阅方式和订阅人，同时支持应用安全负责人查看订阅详情。

### 1.2 用户故事
- **安全管理员**：作为安全管理员，我希望能够管理订阅配置（包括新增、编辑、删除订阅规则），以便根据业务需求灵活配置安全风险的通报策略，实现精准通知。
- **应用安全负责人**：作为应用安全负责人，我希望能够查看订阅详情，了解当前应用的订阅配置情况，以便掌握安全风险的通报对象和方式。

### 1.3 入口来源
- ** 一级模块**：监控告警中心
- **二级模块**：订阅消息管理（新增）
- **三级模块**：订阅管理

### 1.4 前置条件
- 用户需在系统内拥有相应权限
- 安全管理员角色：拥有订阅的增删改查权限
- 应用安全负责人角色：仅拥有订阅的查看权限（只读）

### 1.5 出口去向
- 操作完成后停留在当前页面（订阅管理列表页）
- 支持连续操作（如新增完成后可继续新增）
- 弹窗表单关闭后刷新列表

## 2. 业务流程

### 2.1 流程步骤

| 步骤 | 操作 | 角色 | 说明 |
|------|------|------|------|
| 1 | 进入订阅管理页面 | 安全管理员/应用安全负责人 | 从监控告警中心进入订阅消息管理模块 |
| 2 | 查看订阅列表 | 安全管理员/应用安全负责人 | 展示所有订阅配置记录 |
| 3 | 新增订阅 | 安全管理员 | 点击新增按钮，弹出订阅配置表单 |
| 4 | 填写订阅配置 | 安全管理员 | 填写订阅名称、选择订阅应用、选择订阅方式、选择订阅人 |
| 5 | 保存订阅 | 安全管理员 | 提交表单，创建新的订阅配置 |
| 6 | 编辑订阅 | 安全管理员 | 点击编辑按钮，修改已有订阅配置 |
| 7 | 删除订阅 | 安全管理员 | 点击删除按钮，确认后删除订阅配置 |
| 8 | 查看订阅详情 | 应用安全负责人 | 点击查看按钮，查看订阅配置的详细信息（仅读） |

### 2.2 分支流程

#### 分支一：安全管理员操作流程
```
开始 → 查看列表 → 新增/编辑/删除 → 保存 → 刷新列表 → 结束
```

#### 分支二：应用安全负责人操作流程
```
开始 → 查看列表 → 查看详情 → 关闭详情 → 结束
```

#### 分支三：权限校验流程
```
用户进入页面 → 判断角色权限
├── 安全管理员 → 展示完整操作（新增/编辑/删除/查看）
└── 应用安全负责人 → 仅展示查看按钮
```

### 3.4 关键交互规则

#### 3.4.1 列表页交互

| 交互项 | 规则描述 |
|--------|----------|
| 页面加载 | 默认加载第一页，每页10条数据 |
| 搜索 | 支持即时搜索（输入后自动搜索）或点击搜索按钮 |
| 筛选 | 选择筛选条件后自动过滤列表 |
| 清除筛选 | 点击清除按钮重置所有筛选条件 |
| 排序 | 点击表头可切换升序/降序（默认按创建时间降序） |
| 分页 | 支持跳转页码、调整每页条数 |
| 复选框 | 支持单选、全选、反选 |
| 批量删除 | 选中至少1条后启用批量删除按钮 |
| 无数据 | 展示空状态图和"暂无数据"文案 |

#### 3.4.2 新增/编辑弹窗交互

| 交互项 | 规则描述 |
|--------|----------|
| 弹窗打开 | 从右侧滑入，带遮罩层 |
| 表单校验 | 实时校验，输入时即时提示错误 |
| 订阅人选择 | 弹窗选择，从用户中心加载用户列表 |
| 订阅人搜索 | 支持按姓名、手机号搜索用户 |
| 多选逻辑 | 已选人员展示为标签，点击×移除 |
| 保存提交 | 校验通过后提交，失败提示错误原因 |
| 取消关闭 | 确认取消时提示"确定放弃当前编辑？" |
| 保存成功 | 关闭弹窗，刷新列表，提示"保存成功" |

#### 3.4.3 删除交互

| 交互项 | 规则描述 |
|--------|----------|
| 删除按钮 | 每行右侧操作列显示删除按钮 |
| 删除确认 | 点击后弹出确认框"确定删除该订阅吗？" |
| 批量删除 | 选中多条后点击批量删除，提示"确定删除选中的 X 条订阅吗？" |
| 删除成功 | 刷新列表，提示"删除成功" |
| 删除失败 | 提示错误原因，列表不刷新 |

#### 3.4.4 详情弹窗交互

| 交互项 | 规则描述 |
|--------|----------|
| 查看按钮 | 每行右侧操作列显示查看按钮 |
| 弹窗打开 | 从居中淡入打开 |
| 只读展示 | 所有字段为只读状态，不可编辑 |
| 关闭按钮 | 点击×或点击遮罩层关闭 |
| 关闭确认 | 有未保存更改时提示确认 |

### 3.9 特殊说明

1. **订阅人与通知方式的关系**：订阅配置中的订阅方式为全局配置，每个订阅人收到通知时均使用该方式。若需为不同订阅人配置不同通知方式，需在后续版本中扩展。

2. **订阅应用的继承**：选择"全局"时，该订阅配置适用于所有应用的安全风险告警；选择具体应用时，仅适用于该应用。

3. **订阅人选择器限制**：每次打开订阅人选择器时，重新从用户中心获取最新用户列表，确保人员信息最新。

4. **重复订阅限制**：同一个应用、同一批订阅人、同一订阅方式，仅能创建一条订阅配置（校验重复）。

5. **停用逻辑**：停用后的订阅配置不参与通知发送，但仍保留在列表中，可重新启用。
