# 智能体超市核心页面CodeX适配需求文档（简化版）

**适配工具**：CodeX智能编程工具

**核心目标**：精准生成匹配产品原型的代码，复用标准化业务组件，实现原型视觉效果与业务逻辑1:1还原

**技术栈基础**：React18+TypeScript+Vite+TailwindCSS+MUI/radix-ui（原型导出工程适配）

## 一、整体产品结构

### 1. 产品核心定位

企业内部智能体共享与管理平台，承接上游智能体管理系统的编目/上架数据，同步下游智能体开发平台的授权信息，实现**智能体浏览-申请-审核-授权-使用记录**全流程闭环管理。

### 2. 核心功能模块

```Plain Text

智能体超市系统
├─ 智能体目录页（核心浏览/筛选页）
├─ 申请中心（发起申请/查看申请进度）
├─ 审核中心（处理申请/查看审核历史）
├─ 授权管理（授权配置/续期/回收/查看凭据）
└─ 智能体详情页（查看信息/提交申请/查看授权/使用记录）
```

### 3. 主要角色及操作流程

|角色|核心权限|典型操作流程|
|---|---|---|
|业务用户|浏览智能体、发起使用申请、查看申请进度、查看已授权智能体/使用记录|目录筛选智能体→进入详情→提交申请→查看进度→授权生效后使用|
|审核员|含业务用户权限+处理所属部门申请、驳回/通过申请、填写审核意见|待办审核提醒→查看申请详情→通过/驳回→同步结果给申请人|
|管理员|含审核员权限+智能体上架/下架、批量授权/回收、配置系统参数、查看全局统计|智能体上架审核→批量处理申请→管理全平台授权→查看使用统计|
## 二、核心页面：智能体目录页（主页面）

### 1. 页面核心目标

展示全量已上架智能体，支持**多维度筛选/关键词搜索**，快速定位目标智能体，提供详情页跳转入口，是所有用户的核心操作入口，未登录用户可浏览、登录用户可申请。

### 2. 页面整体布局（从上至下）

整体为**响应式PC端布局**，最大宽度1920px，自适应居中，分5个核心区域，各区域间距24px，均复用标准化业务组件，布局结构如下：

```Plain Text

智能体目录页
├─ 页头区域（PageHeader）：页面标题+面包屑
├─ 筛选区域（FilterBar）：搜索框+标签筛选+高级筛选面板
├─ 列表区域（CardList）：3列栅格卡片式列表（屏幕<1440px自动切2列）
├─ 空状态区域（Empty）：无数据时展示（复用空状态业务组件）
└─ 分页区域（Pagination）：分页控件+总条数展示
```

### 3. 各区域交互操作与业务逻辑

#### 3.1 页头区域

- **复用组件**：通用页头业务组件

- **核心元素**：页面标题「智能体目录」+ 面包屑「首页>智能体目录」

- **交互逻辑**：面包屑可点击跳转，标题无交互，仅做页面标识

#### 3.2 筛选区域（核心交互区）

- **复用组件**：带搜索的下拉选择器、标签化筛选栏、高级筛选面板（均为标准化业务组件）

- **核心元素**：关键词搜索框+固定标签筛选+高级筛选（共享方式/建设单位/排序）

- **交互&业务逻辑**：

    1. 搜索框：输入防抖500ms，回车/点击搜索图标触发，支持清空，无结果显示空状态；

    2. 标签筛选：单选模式，点击「全部」重置所有标签，筛选后实时更新列表；

    3. 高级筛选：下拉选择后自动触发筛选，无需额外点击，筛选条件本地记忆（刷新不丢失）；

    4. 所有筛选操作**无刷新更新列表**，加载中展示骨架屏（复用加载状态业务组件）。

#### 3.3 列表区域（核心展示区）

- **复用组件**：卡片式列表业务组件、脱敏展示组件、展开收起文本组件

- **核心元素**：智能体卡片（单卡片为最小单元，批量展示）

- **交互&业务逻辑**：

    1. 卡片交互：整卡可点击，跳转至对应智能体详情页；hover时卡片阴影加深+轻微上浮；

    2. 数据展示：简介超出3行自动折叠，hover无脱敏，敏感信息（若有）按规则脱敏展示；

    3. 权限控制：未登录用户点击卡片→跳转登录页，登录用户→直接跳转详情页；

    4. 懒加载：卡片图标/图片进入视口才加载，加载失败显示默认占位图。

#### 3.4 空状态区域

- **复用组件**：空状态处理业务组件

- **核心元素**：无数据图标+文案「暂无符合条件的智能体」+ 引导按钮「去查看申请指南」

- **交互逻辑**：引导按钮点击跳转至系统帮助页，无其他交互。

#### 3.5 分页区域

- **复用组件**：分页保留选中业务组件（复用分页逻辑）

- **核心元素**：页码按钮+上/下一页+总条数文字「共找到X个智能体」

- **交互&业务逻辑**：

    1. 分页参数：每页9条（3列×3行），点击页码无刷新更新列表；

    2. 筛选联动：任意筛选条件变更，页码自动重置为第1页；

    3. 状态记忆：当前页码本地存储，刷新页面保留。

### 5. 页面通用规则

1. **权限控制**：未登录用户可浏览/筛选，不可跳转详情；管理员额外显示「批量操作」按钮（批量上架/下架）；

2. **响应式适配**：支持1366×768/1440×900/1920×1080主流分辨率，无横向滚动条；

3. **性能要求**：首屏加载≤2s，筛选响应≤500ms，所有图标/图片懒加载；

4. **浏览器兼容**：Chrome≥80、Firefox≥75、Edge≥80、Safari≥14，无样式错乱。

3. 若需扩展组件功能，先更新标准化组件库，再引用，禁止在页面中修改组件源码。

## 四、CodeX生成核心要求

1. **代码可直接运行**：生成后执行`npm install && npm run dev`可正常启动，无编译/运行错误；

2. **视觉1:1还原**：所有尺寸、颜色、间距、布局严格遵循本文档规范，与产品原型一致；

3. **交互逻辑完整**：所有筛选、跳转、分页、权限控制逻辑与本文档一致，无遗漏；

4. **组件复用达标**：必须引用指定标准化业务组件，无重复开发的组件代码；

5. **数据内置mock**：内置至少9条智能体mock数据，覆盖所有状态（已授权/申请中/未申请），无需依赖后端接口即可展示完整页面。
